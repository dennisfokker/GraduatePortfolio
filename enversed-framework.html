enversed's framework

Enversed has created a framework which contains some boilerplate code which is used in virtually every project. Aside from that, the framework also contains modules, which are extensions onto the previously mentioned boilerplate.

While the framework as a whole was already in place, there were several additions I have worked on.

hand collision

I designed a system used to make a visual representation of your hand physical, whilst minimizing the chance of breaking the immersion of the game. This is based of information gathered from research, developers and input from the VR-community

It’s a visualisation that’s an extension on Enversed’s human pawn module. It contains configuration settings (such as turning it off), but for the rest should work out-of-the-box.

HAND COLLISION

A system to make hands physical in the VR world. Could be used to prevent sequence-breaking through walls.

screenshot tool

Whilst the basis of a screenshot tool was already there, I made the extensions so screenshots can be made from any source (either another player or a special type of actor).

Furthermore the screenshot tool now also allows for previewing on a render target, whilst keeping in mind the graphical load this would provide. In order to minimize this the preview generally makes sure not to render more than it needs to, but also contains failsafes (such as making sure the programmer can’t (accidentally) start up multiple preview without deliberately creating multiple tools).

SCREENSHOT TOOL

A components that allow users to take screenshots from specific sources and preview what they would look like.
