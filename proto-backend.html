arduino communication

Some of the games in the PROTO game collection use input from physical machines. These machines work with a microcontroller (e.g. Arduino) which can communicate with the game through a plugin.

Communication between the game and the hardware was defined with a protocol and the full list of tasks to implement the client’s initial design required were listen (and later adjusted based on client input).

license spring

Not all systems are viable to be made in-house. Licensing is currently one of them. License Spring is the chosen framework to handle the licensing at Enversed Studios.

In order the implement License Spring’s SDK into UE4, a plugin was created that mostly acts as a Façade. It simplifies communication by hiding certain parts of the implementation that should be static behind the plugin’s code (such as checking if there’s a valid license active), whilst allowing any actions a project might require to be custom made.

LICENSE SPRING

Creating a UE4 plugin that exposes a third-party license library and the issues encountered during development.
