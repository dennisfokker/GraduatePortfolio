proto back-end

The PROTO Invention Factory is a client who requested several games from Enversed, most of which I have worked on. Some systems were required for multiple games. You can find these “back-end” products listed here.

arduino communication

Some of the games in the PROTO game collection use input from physical machines. These machines have their hardware’s data run through an Arduino and from there over to the PC that runs the game.

This communication can also go backwards, where the game communicates information with the Arduino.

ARDUINO COMMUNICATION

The communication between an Arduino's connected hardware to the game to interact with the game.

license spring

Not all systems are viable to be made in-house. Licensing is currently one of them. License Spring is the chosen framework to handle the licensing at Enversed Studios. The only thing we have to do is to implement it into Unreal Engine 4.

While licensing is going to be used in future projects too, it’s first implementation was for PROTO.

LICENSE SPRING

Creating a UE4 plugin that exposes a third-party license library and the issues encountered during development.
